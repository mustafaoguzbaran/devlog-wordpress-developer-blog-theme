RESTful API Tasarımında En İyi Pratikler

Giriş

Modern web uygulamalarında API tasarımı, uygulamanın başarısını doğrudan etkileyen kritik bir faktördür. RESTful API'ler, basitlikleri ve standartlaşmış yapıları sayesinde en popüler API mimarisi haline gelmiştir.

Bu yazıda, REST API tasarımında dikkat edilmesi gereken en önemli noktaları, yaygın hataları ve en iyi pratikleri detaylı bir şekilde inceleyeceğiz.

Not: Bu rehber, orta ve ileri seviye backend geliştiriciler için hazırlanmıştır. Temel HTTP ve web geliştirme bilgisi varsayılmaktadır.

REST Prensipleri

REST (Representational State Transfer), Roy Fielding tarafından 2000 yılında tanımlanan bir mimari stildir. REST'in altı temel prensibi vardır:

  1. Client-Server: İstemci ve sunucu birbirinden bağımsız olmalıdır
  2. Stateless: Her istek, ihtiyaç duyduğu tüm bilgiyi içermelidir
  3. Cacheable: Yanıtlar önbelleğe alınabilir olmalıdır
  4. Uniform Interface: Tutarlı arayüz kullanılmalıdır
  5. Layered System: Katmanlı sistem mimarisi desteklenmelidir
  6. Code on Demand: İsteğe bağlı kod indirimi (opsiyonel)

HTTP Metodları

RESTful API'lerde HTTP metodları, gerçekleştirilmek istenen işlemin türünü belirtir. Her metodun kendine özgü anlamı ve kullanım alanı vardır:

HTTP Methods 
GET    /api/users          # Tüm kullanıcıları listele
GET    /api/users/123      # ID'si 123 olan kullanıcıyı getir
POST   /api/users          # Yeni kullanıcı oluştur
PUT    /api/users/123      # ID'si 123 olan kullanıcıyı güncelle
PATCH  /api/users/123      # ID'si 123 olan kullanıcıyı kısmen güncelle
DELETE /api/users/123      # ID'si 123 olan kullanıcıyı sil

GET Metodu

GET metodu, sunucudan veri almak için kullanılır. Bu metod idempotent (tekrarlanabilir) ve safe (güvenli) olmalıdır, yani sunucu durumunda herhangi bir değişiklik yapmamalıdır.

POST Metodu

POST metodu, yeni kaynak oluşturmak için kullanılır. Bu metod ne idempotent ne de safe'dir, çünkü her çağrıda yeni bir kaynak oluşturur.

HTTP Status Kodları

Doğru HTTP status kodlarının kullanımı, API'nin kullanılabilirliği açısından kritik öneme sahiptir. İşte en sık kullanılan status kodları:

Kod Anlamı Kullanım Alanı
200 OK Başarılı GET, PUT, PATCH istekleri
201 Created Başarılı POST istekleri
204 No Content Başarılı DELETE istekleri
400 Bad Request Geçersiz istek formatı
401 Unauthorized Kimlik doğrulama gerekli
404 Not Found Kaynak bulunamadı
500 Internal Server Error Sunucu hatası

Endpoint Tasarımı

İyi tasarlanmış endpoint'ler, API'nin anlaşılırlığını ve kullanılabilirliğini önemli ölçüde artırır. İşte temel kurallar:

Kaynak Odaklı Tasarım

URL'ler kaynakları (resources) temsil etmeli, fiilleri (verbs) değil. HTTP metodları zaten eylemi belirtir.

İyi Örnekler

  • GET /api/users
  • POST /api/products
  • PUT /api/orders/123

Kötü Örnekler

  • GET /api/getUsers
  • POST /api/createProduct
  • PUT /api/updateOrder/123

Hiyerarşik İlişkiler

Kaynaklar arasındaki ilişkileri URL yapısında göstermek, API'yi daha anlaşılır kılar:

URL Hierarchy 
GET /api/users/123/posts           # Kullanıcının gönderileri
GET /api/posts/456/comments        # Gönderinin yorumları
GET /api/categories/789/products   # Kategorideki ürünler

API Versiyonlama

API versiyonlama, geriye dönük uyumluluğu korurken yeni özellikler eklemek için kritik öneme sahiptir. En yaygın yaklaşımlar:

URL Path Versioning

URL Versioning 
GET /api/v1/users
GET /api/v2/users

Header Versioning

Header Versioning 
GET /api/users
Accept: application/vnd.api+json;version=1

Güvenlik Konuları

API güvenliği, modern web uygulamalarının en kritik aspectlerinden biridir. İşte temel güvenlik önlemleri:

Kimlik Doğrulama

  • JWT Token: Stateless kimlik doğrulama
  • OAuth 2.0: Üçüncü parti erişim kontrolü
  • API Keys: Basit erişim kontrolü

HTTPS Kullanımı

Tüm API endpoint'leri HTTPS üzerinden sunulmalıdır. Bu, veri iletiminin şifrelenmesini ve man-in-the-middle saldırılarından korunmayı sağlar.

Uyarı: Asla hassas bilgileri (şifreler, API anahtarları) URL parametrelerinde göndermeyin. Bu bilgiler server loglarında görünebilir.

Sonuç

RESTful API tasarımı, modern web geliştirmede vazgeçilmez bir beceridir. Bu yazıda ele aldığımız prensipler ve en iyi pratikler, hem geliştirici deneyimini hem de API performansını önemli ölçüde iyileştirecektir.

Unutmayın ki, iyi bir API tasarımı sadece teknik gereksinimleri karşılamakla kalmaz, aynı zamanda geliştiricilerin işini kolaylaştırır ve projenizin uzun vadeli başarısına katkıda bulunur.
